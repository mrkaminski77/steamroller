# Jobs

Synapse Spark jobs are defined and deployed from the `jobs/` directory.

## Directory structure

Each job lives in its own subdirectory named after the job:

```
jobs/
    job.json               # shared Spark job definition template
    deploy-job.ps1         # deployment script
    <job_name>/
        <job_name>.py      # PySpark entry point
        <job_name>.json    # generated job definition (do not edit, do not commit)
```

The `<job_name>.json` file is generated by `deploy-job.ps1` and should be excluded from source control.

## Python entry point conventions

The entry point script must:

- Be named `<job_name>.py` and placed in `jobs/<job_name>/`
- Accept all runtime inputs as command-line arguments (no hardcoded paths or config)
- Use `argparse` to parse arguments
- Pass complex objects (arrays, objects) as **base64-encoded JSON strings**

## Deploying a job

Run `deploy-job.ps1` from the `jobs/` directory:

```powershell
Set-Location jobs

.\deploy-job.ps1 `
    -synapse_workspace <workspace_name> `
    -storage_account   <storage_account_name> `
    -storage_container <container_name> `
    -job_name          <job_name> `
    -spark_pool        <spark_pool_name>
```

The script will:

1. Read `job.json` as a template
2. Set `targetBigDataPool.referenceName` to `$spark_pool`
3. Set `jobProperties.file` to the ABFSS path of the uploaded script
4. Write the resolved definition to `<job_name>/<job_name>.json`
5. Upload `<job_name>/<job_name>.py` to `scripts/<job_name>.py` in the storage container
6. Create or update the Spark Job Definition in Synapse under the `Ingestion` folder

**Prerequisites:** Azure CLI installed and logged in with access to the target Synapse workspace and storage account.

## Spark pool configuration

Compute defaults are defined in `job.json` and apply to all jobs:

| Setting | Default |
|---|---|
| Spark version | 3.4 |
| Executors | 2 |
| Executor cores | 4 |
| Executor memory | 28g |
| Driver cores | 4 |
| Driver memory | 28g |

To use different compute for a specific job, edit the relevant values in the generated `<job_name>.json` and re-run the deploy script (or update the job definition directly in Synapse Studio).
